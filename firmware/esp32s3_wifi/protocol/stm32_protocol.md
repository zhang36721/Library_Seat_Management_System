# STM32 通信协议

## 协议概述

本协议用于 ESP32S3 与主控 STM32 之间的串行通信。

## 物理层

- 接口：UART
- 波特率：115200
- 数据位：8
- 停止位：1
- 校验位：无

## 帧格式

### 标准帧
```
+------+------+---------+------+
| 0xFF | Len  | Data... | 0xFF |
+------+------+---------+------+
  帧头   长度    数据    帧尾
```

### 帧字段说明

| 字段 | 长度（字节） | 说明 |
|------|-------------|------|
| 帧头 | 1 | 固定为 0xFF |
| 长度 | 1 | 数据字段的长度 |
| 数据 | 可变 | 实际数据内容 |
| 帧尾 | 1 | 固定为 0xFF |

## 命令定义

### 命令列表

| 命令码 | 命令名称 | 数据长度 | 说明 |
|--------|----------|---------|------|
| 0x01 | SEAT_STATUS | 5 | 座位状态更新 |
| 0x02 | CARD_EVENT | 18 | 刷卡事件 |
| 0x03 | SYSTEM_INFO | 可变 | 系统信息 |
| 0x04 | HEARTBEAT | 0 | 心跳包 |
| 0x05 | ACK | 1 | 确认响应 |
| 0x06 | NACK | 1 | 否定响应 |

### SEAT_STATUS（0x01）

**数据结构**:
```
+------+------+------+------+------+------+
| 0x01 | SeatID | Status | Timestamp (4字节) |
+------+------+------+------+------+------+
```

**字段说明**:
- SeatID: 座位编号（1-3）
- Status: 座位状态（0=空闲, 1=占用）
- Timestamp: Unix 时间戳（大端序）

### CARD_EVENT（0x02）

**数据结构**:
```
+------+------+------+------+------+------+------+------+------+------+
| 0x02 | UID[0] | UID[1] | UID[2] | UID[3] | EventType | SeatID | Timestamp |
+------+------+------+------+------+------+------+------+------+------+
```

**字段说明**:
- UID[0-3]: 卡的唯一标识符
- EventType: 事件类型（0=进入, 1=离开）
- SeatID: 座位编号（0表示无座位）
- Timestamp: Unix 时间戳（大端序）

### HEARTBEAT（0x04）

**数据结构**:
```
+------+
| 0x04 |
+------+
```

### ACK（0x05）

**数据结构**:
```
+------+------+
| 0x05 | Cmd  |
+------+------+
```

**字段说明**:
- Cmd: 被确认的命令码

### NACK（0x06）

**数据结构**:
```
+------+------+------+
| 0x06 | Cmd  | ErrCode |
+------+------+------+
```

**字段说明**:
- Cmd: 失败的命令码
- ErrCode: 错误码

## 错误码

| 错误码 | 说明 |
|--------|------|
| 0x00 | 成功 |
| 0x01 | 无效命令 |
| 0x02 | 数据长度错误 |
| 0x03 | 校验失败 |
| 0x04 | 超时 |

## 通信流程

### 初始化阶段
1. ESP32S3 发送 HEARTBEAT
2. STM32 回复 ACK
3. 建立通信连接

### 数据传输阶段
1. STM32 发送数据帧
2. ESP32S3 解析并确认
3. ESP32S3 上传数据到服务器
4. ESP32S3 发送 ACK/NACK

### 异常处理
- 超时重传
- 帧错误处理
- 连接恢复

## 示例帧

### 座位状态帧
```
FF 05 01 01 01 5E A7 8C 00 FF
FF: 帧头
05: 数据长度
01: 命令码 SEAT_STATUS
01: 座位ID
01: 状态（占用）
5E A7 8C 00: 时间戳
FF: 帧尾
```

### 刷卡事件帧
```
FF 12 02 AA BB CC DD 00 01 5E A7 8C 00 FF
FF: 帧头
12: 数据长度（18字节）
02: 命令码 CARD_EVENT
AA BB CC DD: 卡UID
00: 事件类型（进入）
01: 座位ID
5E A7 8C 00: 时间戳
FF: 帧尾
```

### 心跳帧
```
FF 01 04 FF
FF: 帧头
01: 数据长度
04: 命令码 HEARTBEAT
FF: 帧尾
```
