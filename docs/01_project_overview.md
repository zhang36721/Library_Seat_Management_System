# 项目概述

## 1. 项目背景

图书馆座位资源紧张，占座现象严重，需要一套智能化的座位管理系统来提高座位利用率。

## 2. 项目目标

- 实时监测座位占用状态
- 提供座位预约和签到功能
- 自动化管理，减少人工干预
- 提供数据统计和分析功能

## 3. 系统架构

```
┌─────────────────┐    ZigBee    ┌─────────────────┐
│  座位检测节点   │◄────────────►│   主控节点      │
│  (STM32+ZigBee) │  无线通信    │  (STM32+ZigBee) │
└─────────────────┘              └────────┬────────┘
                                          │
                                          │ UART
                                          ▼
                                   ┌─────────────────┐
                                   │   ESP32S3       │
                                   │   WiFi 网关     │
                                   └────────┬────────┘
                                            │
                                            │ HTTP
                                            ▼
                                   ┌─────────────────┐
                                   │   Python 后端   │
                                   │   数据库        │
                                   └────────┬────────┘
                                            │
                                            │ API
                                            ▼
                                   ┌─────────────────┐
                                   │   Vue3 前端     │
                                   │   管理界面      │
                                   └─────────────────┘
```

## 4. 功能需求

### 4.1 座位检测
- 红外/压力传感器检测座位是否有人
- 定时上报状态到主控节点
- 低功耗设计，电池供电

### 4.2 数据收集
- 主控节点收集所有座位节点数据
- 数据汇总后通过 ESP32S3 上传
- 断网缓存，网络恢复后补传

### 4.3 后端服务
- RESTful API 接口
- 用户管理和权限控制
- 座位状态管理和统计
- 预约和签到功能

### 4.4 前端界面
- 座位状态可视化
- 管理员后台
- 用户预约界面
- 数据统计图表

## 5. 非功能需求

- 系统响应时间 < 2秒
- 座位检测准确率 > 95%
- 节点电池续航 > 6个月
- 支持至少 500 个座位节点
