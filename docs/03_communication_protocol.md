# 通信协议定义

本文档定义系统中各模块之间的通信协议和数据格式。系统按 3 个座位设计。

## 1. 座位检测端 STM32 → ZigBee → 主控端 STM32

### 1.1 通信方式
- **物理层**: ZigBee 无线通信（CC2530 模块）
- **传输层**: 串口透传模式
- **波特率**: 9600 bps
- **数据位**: 8
- **停止位**: 1
- **校验位**: 无

### 1.2 数据帧格式

| 字段 | 长度(字节) | 说明 |
|------|------------|------|
| 帧头 | 2 | 0xAA 0x55 |
| 节点ID | 1 | 座位节点编号（1/2/3） |
| 命令字 | 1 | 0x01:状态上报 0x02:心跳包 |
| 数据长度 | 1 | 后续数据字节数 |
| 数据区 | N | 具体数据内容 |
| 校验和 | 1 | 前面所有字节的异或校验 |

### 1.3 座位状态数据格式

| 状态码 | 含义 |
|--------|------|
| 0x00 | 空闲 |
| 0x01 | 占用 |
| 0x02 | 疑似占用 / 异常 |

### 1.4 数据区格式（状态上报）

| 字段 | 长度(字节) | 说明 |
|------|------------|------|
| 红外状态 | 1 | 0=无人 1=有人 |
| 压力状态 | 1 | 0=无压力 1=有压力 |
| 综合状态 | 1 | 0=空闲 1=占用 2=疑似占用 |
| RSSI | 1 | 信号强度 |

### 1.5 通信流程
1. 座位节点定时（如每 5 秒）上报状态
2. 主控节点收到后回复 ACK
3. 未收到 ACK 则重传（最多 3 次）

## 2. 主控端 STM32 → UART → ESP32S3

### 2.1 通信方式
- **物理层**: UART 串口
- **波特率**: 115200 bps
- **数据位**: 8
- **停止位**: 1
- **校验位**: 无

### 2.2 数据帧格式

| 字段 | 长度(字节) | 说明 |
|------|------------|------|
| 帧头 | 2 | 0xBB 0x66 |
| 命令字 | 1 | 0x10:数据上传 0x11:心跳 |
| 数据长度 | 2 | 后续数据字节数(大端序) |
| 数据区 | N | 具体数据内容 |
| CRC16 | 2 | CRC16-Modbus 校验 |

### 2.3 数据区格式（命令字 0x10）

| 字段 | 长度(字节) | 说明 |
|------|------------|------|
| device_id | 2 | 设备编号 |
| time | 4 | Unix 时间戳 |
| seat1 | 1 | 座位 1 状态（0/1/2） |
| seat2 | 1 | 座位 2 状态（0/1/2） |
| seat3 | 1 | 座位 3 状态（0/1/2） |
| free_seats | 1 | 空闲座位数 |
| card_id | 6 | 当前刷卡 ID（未刷卡为 0） |
| action | 1 | 动作类型（0=无 1=签到 2=签退） |
| gate_status | 1 | 闸机状态（0=关 1=开） |

### 2.4 通信流程
1. 主控节点定时（如每 5 秒）汇总数据
2. ESP32S3 收到后存入缓存队列
3. ESP32S3 通过网络上传到服务器

## 3. ESP32S3 → HTTP POST → Python FastAPI 后端

### 3.1 通信方式
- **协议**: HTTP
- **方法**: POST
- **数据格式**: JSON
- **Content-Type**: application/json

### 3.2 数据上传接口

**URL**: `/api/v1/seat/status/upload`

**请求体**:
```json
{
  "device_id": 1,
  "time": 1698765432,
  "seat1": 1,
  "seat2": 0,
  "seat3": 0,
  "free_seats": 2,
  "card_id": "A1B2C3D4E5",
  "action": 1,
  "gate_status": 0
}
```

**字段说明**:
| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | int | 设备编号 |
| time | int | Unix 时间戳 |
| seat1 | int | 座位 1 状态（0=空闲 1=占用 2=疑似占用） |
| seat2 | int | 座位 2 状态（0=空闲 1=占用 2=疑似占用） |
| seat3 | int | 座位 3 状态（0=空闲 1=占用 2=疑似占用） |
| free_seats | int | 空闲座位数 |
| card_id | string | 当前刷卡 ID（未刷卡为空字符串） |
| action | int | 动作类型（0=无 1=签到 2=签退） |
| gate_status | int | 闸机状态（0=关 1=开） |

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "received": true,
    "timestamp": 1698765432
  }
}
```

### 3.3 心跳接口

**URL**: `/api/v1/device/heartbeat`

**请求体**:
```json
{
  "device_id": 1,
  "timestamp": 1698765432,
  "status": "online"
}
```

### 3.4 通信流程
1. ESP32S3 定时上传数据（如每 5 秒）
2. 服务器返回确认响应
3. 上传失败则重试（指数退避，最多 3 次）
4. 定时发送心跳保持连接（如每 30 秒）
