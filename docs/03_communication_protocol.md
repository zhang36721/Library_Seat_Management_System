# 通信协议定义

本文档定义系统中各模块之间的通信协议和数据格式。

## 1. 座位检测端 STM32 与主控端 STM32：ZigBee 串口透传

### 1.1 通信方式
- **物理层**: ZigBee 无线通信（CC2530 模块）
- **传输层**: 串口透传模式
- **波特率**: 9600 bps
- **数据位**: 8
- **停止位**: 1
- **校验位**: 无

### 1.2 数据帧格式

| 字段 | 长度(字节) | 说明 |
|------|------------|------|
| 帧头 | 2 | 0xAA 0x55 |
| 节点ID | 2 | 座位节点唯一标识 |
| 命令字 | 1 | 0x01:状态上报 0x02:心跳包 0x03:查询响应 |
| 数据长度 | 1 | 后续数据字节数 |
| 数据区 | N | 具体数据内容 |
| 校验和 | 1 | 前面所有字节的异或校验 |

### 1.3 座位状态数据格式

| 状态码 | 含义 |
|--------|------|
| 0x00 | 空闲 |
| 0x01 | 占用 |
| 0x02 | 预约 |
| 0xFF | 异常 |

### 1.4 通信流程
1. 座位节点定时（如每30秒）上报状态
2. 主控节点收到后回复 ACK
3. 未收到 ACK 则重传（最多3次）

## 2. 主控端 STM32 与 ESP32S3：UART 串口通信

### 2.1 通信方式
- **物理层**: UART 串口
- **波特率**: 115200 bps
- **数据位**: 8
- **停止位**: 1
- **校验位**: 无

### 2.2 数据帧格式

| 字段 | 长度(字节) | 说明 |
|------|------------|------|
| 帧头 | 2 | 0xBB 0x66 |
| 命令字 | 1 | 0x10:数据上传 0x11:心跳 0x12:配置 |
| 数据长度 | 2 | 后续数据字节数(大端序) |
| 数据区 | N | 具体数据内容 |
| CRC16 | 2 | CRC16-Modbus 校验 |

### 2.3 批量座位数据格式

| 字段 | 长度(字节) | 说明 |
|------|------------|------|
| 时间戳 | 4 | Unix 时间戳 |
| 节点数量 | 2 | 本次上报的节点数 |
| 节点数据 | N×4 | 每个节点：ID(2) + 状态(1) + 信号强度(1) |

### 2.4 通信流程
1. 主控节点定时（如每10秒）汇总数据
2. ESP32S3 收到后存入缓存队列
3. ESP32S3 通过网络上传到服务器

## 3. ESP32S3 与 Python 后端：HTTP POST 上传数据

### 3.1 通信方式
- **协议**: HTTP/HTTPS
- **方法**: POST
- **数据格式**: JSON
- **Content-Type**: application/json

### 3.2 数据上传接口

**URL**: `/api/v1/seat/status/batch`

**请求体**:
```json
{
  "device_id": "esp32s3_001",
  "timestamp": 1698765432,
  "seats": [
    {
      "seat_id": "A001",
      "status": 1,
      "rssi": -65
    },
    {
      "seat_id": "A002",
      "status": 0,
      "rssi": -70
    }
  ]
}
```

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "received": 2,
    "processed": 2
  }
}
```

### 3.3 心跳接口

**URL**: `/api/v1/device/heartbeat`

**请求体**:
```json
{
  "device_id": "esp32s3_001",
  "timestamp": 1698765432,
  "status": "online"
}
```

### 3.4 通信流程
1. ESP32S3 定时上传数据（如每10秒）
2. 服务器返回确认响应
3. 上传失败则重试（指数退避）
4. 定时发送心跳保持连接
