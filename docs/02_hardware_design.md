# 硬件设计说明

## 1. 系统架构

本系统由两个 STM32 节点组成：

- **座位检测端 STM32**：统一采集 3 个座位的红外和压力传感器数据，通过 ZigBee 发送
- **主控管理端 STM32**：接收 ZigBee 数据，负责 OLED 显示、DS1302 时钟、RC522 刷卡、步进电机控制、蜂鸣器提醒、按键输入和 ESP32S3 串口通信

## 2. 座位检测端

### 2.1 核心组件
- **MCU**: STM32F103C8T6
- **无线模块**: CC2530 ZigBee 串口透传模块（发送端）
- **传感器**: 3 路红外感应模块 + 重量压力传感器 + HX711AD
- **电源**: 5V USB 供电

### 2.2 设计要点
- 统一采集 3 个座位的传感器数据
- 定时通过 ZigBee 发送座位状态到主控端
- 每个座位包含红外和压力双重检测

## 3. 主控管理端

### 3.1 核心组件
- **MCU**: STM32F103C8T6
- **无线模块**: CC2530 ZigBee 串口透传模块（接收端）
- **显示**: 0.96 寸 OLED IIC 屏
- **时钟**: DS1302 时钟模块
- **刷卡**: RC522 RFID 模块
- **闸机**: 28BYJ-48 步进电机 + ULN2003 驱动板
- **提醒**: 有源蜂鸣器模块
- **输入**: 独立按键模块（4~6 个）
- **通信**: ESP32S3（UART 串口通信）
- **电源**: 5V USB 供电

### 3.2 设计要点
- 通过 ZigBee 接收座位检测端数据
- OLED 实时显示时间、座位状态、刷卡信息
- RC522 识别用户 IC 卡
- 刷卡成功后控制步进电机模拟闸机开关
- 蜂鸣器提供声音提醒
- 按键用于时间设置、参数调整、卡片管理

## 4. WiFi 网关

### 4.1 核心组件
- **MCU**: ESP32S3
- **通信接口**: UART 转主控 STM32
- **网络**: WiFi 2.4GHz
- **电源**: 5V USB 供电

### 4.2 设计要点
- 通过 UART 接收主控端数据
- WiFi 自动重连
- HTTP POST 上传数据到 Python FastAPI 后端
- 断网数据缓存
